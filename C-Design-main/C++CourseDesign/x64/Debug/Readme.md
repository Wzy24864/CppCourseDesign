# 图书管理系统使用说明

## 一、系统概述

  本系统是一个基于C++的图书管理程序，支持图书信息的增删改查、借书还书管理等功能。数据采用文件持久化存储，提供手动保存和退出自动保存两种存盘方式。

------

## 二、文件结构

  | 文件名                              | 作用                                   |
  | :---------------------------------- | :------------------------------------- |
  | `main.cpp`                          | 程序入口，显示菜单并响应用户操作       |
  | `System.h / System.cpp`             | 系统核心类，实现所有业务逻辑和文件读写 |
  | `Book.h / Book.cpp`                 | 图书类，存储图书基本信息               |
  | `Student.h / Student.cpp`           | 学生类，存储借书人信息                 |
  | `BorrowRecord.h / BorrowRecord.cpp` | 借阅记录类，存储借阅相关信息           |
  | `Times.h / Time.cpp`                | 时间类，存储日期信息                   |
  | `books.txt`                         | 图书数据文件（自动生成）               |
  | `students.txt`                      | 学生数据文件（自动生成）               |
  | `records.txt`                       | 借阅记录文件（自动生成）               |
  | `Readme.md`                         | 帮助文档（本文件）                     |

------

## 三、功能说明

  ### 主菜单

  text

  ```
  ===== 欢迎使用图书管理系统! =====
  1.新进图书信息的录入
  2.更改图书信息
  3.办理借书手续
  4.办理还书手续
  5.删除借阅信息
  6.保存文件
  7.查看帮助文档
  8.退出程序
  请输入数字以继续:
  ```

  

------

  ### 功能1：新进图书信息的录入

  **函数：** `void System::newBookEnter()`

  **作用：** 向系统中添加新图书，信息暂存于内存中。

  **操作流程：**

  1. 输入书名
  2. 输入图书编号（唯一标识）
  3. 输入价格
  4. 输入作者
  5. 输入库存数量

  **输入示例：**

  text

  ```
  书名: C++程序设计
  编号: ISBN-001
  价格: 59.9
  作者: 张三
  库存: 10
  ```

  

------

  ### 功能2：图书信息的查询、修改和删除

  **函数：** `void System::manageBook()`

  **作用：** 根据书名或编号搜索图书，然后进行修改或删除操作。

  **操作流程：**

  1. 输入搜索关键词（支持书名或编号的模糊匹配）
  2. 系统显示匹配的图书列表
  3. 选择操作类型：
     - `1` — 修改图书信息
     - `2` — 删除图书
     - `0` — 返回主菜单
  4. 选择要操作的图书序号

  **修改操作说明：**

  - 可修改书名、价格、作者
  - 按回车跳过表示不修改该项
  - 输入负数跳过价格修改

------

  ### 功能3：办理借书手续

  **函数：** `void System::borrowBook()`

  **作用：** 录入借阅信息，借出图书，自动减少库存。

  **操作流程：**

  1. 输入图书编号
  2. 系统检查图书是否存在、库存是否充足
  3. 输入借书人姓名和学号
  4. 输入借出日期和应还日期
  5. 系统自动减少图书库存

  **约束条件：**

  - 图书必须已录入系统
  - 图书库存必须大于0
  - 学号不存在时会自动创建新学生记录

  **输入示例：**

  text

  ```
  图书编号: ISBN-001
  借书人姓名: 李四
  学号: 2024001
  借出日期(年 月 日): 2026 4 29
  应还日期(年 月 日): 2026 5 29
  ```

  

------

  ### 功能4：办理还书手续

  **函数：** `void System::returnBook()`

  **作用：** 归还已借图书，更新借阅状态并恢复库存。

  **操作流程：**

  1. 输入图书编号
  2. 输入学号
  3. 系统查找对应的借阅记录
  4. 将借阅状态改为"已归还（Returned）"
  5. 自动恢复图书库存

  **约束条件：**

  - 必须存在对应的借阅记录
  - 借阅状态必须为"借出中（LEND）"

------

  ### 功能5：删除借阅信息

  **函数：** `void System::deleteRecord()`

  **作用：** 从系统中删除指定的借阅记录。

  **操作流程：**

  1. 输入图书编号
  2. 输入学号
  3. 系统查找并删除对应的借阅记录

  >  **注意：** 未归还的信息不能被删除！(除非手动删除数据，但这将会导致图书数目出现异常)

------

  ### 功能6：保存文件

  **函数：** `void System::save()`

  **作用：** 手动将内存中的全部数据立即写入磁盘文件。

  **调用时机：**

  - 用户主动选择菜单选项6
  - 程序正常退出时（选项8）自动调用

  **保存的文件：**

  - `books.txt` — 所有图书信息
  - `students.txt` — 所有学生信息
  - `records.txt` — 所有借阅记录

------

  ### 功能7：查看帮助文档

  **选项：** 菜单第7项

  **代码：** `system("start Readme.md"); // 通过Windows CMD 打开帮助文档`

  **作用：** 调用系统命令打开 `Readme.md` 文件，查看本帮助文档。

------

  ### 功能8：退出程序

  **选项：** 菜单第8项

  **作用：** 自动保存所有数据并正常退出程序。

------

## 四、文件读写函数说明

  ### 数据加载函数（构造函数自动调用）

  | 函数                  | 作用                         | 读取文件       |
  | :-------------------- | :--------------------------- | :------------- |
  | `void loadBooks()`    | 从文件加载所有图书信息到内存 | `books.txt`    |
  | `void loadStudents()` | 从文件加载所有学生信息到内存 | `students.txt` |
  | `void loadRecords()`  | 从文件加载所有借阅记录到内存 | `records.txt`  |

  ### 数据保存函数

  | 函数                  | 作用         | 写入文件       | 调用方式                   |
  | :-------------------- | :----------- | :------------- | :------------------------- |
  | `void saveBooks()`    | 保存图书信息 | `books.txt`    | 由 `save()` 调用           |
  | `void saveStudents()` | 保存学生信息 | `students.txt` | 由 `save()` 调用           |
  | `void saveRecords()`  | 保存借阅记录 | `records.txt`  | 由 `save()` 调用           |
  | `void save()`         | 统一保存接口 | 以上全部       | 菜单选项6 / 退出时自动调用 |

## 五、数据文件格式

  ### books.txt

  text

  ```
  书名
  编号
  价格
  作者
  库存
  ```

  

  每本图书占5行，依次排列。

  ### students.txt

  text

  ```
  姓名
  学号
  ```

  

  每个学生占2行，依次排列。

  ### records.txt

  text

  ```
  图书编号
  借书人姓名
  学生姓名
  学号
  借出年 借出月 借出日
  应还年 应还月 应还日
  状态值
  ```

  

  每条记录占7行，依次排列。

## 六、借阅状态说明

  | 状态值 | 枚举名     | 含义     |
  | :----- | :--------- | :------- |
  | 1      | `LEND`     | 借出中   |
  | 0      | `RETURN`   | 已归还   |
  | -1     | `OVERTIME` | 超期未还 |

## 七、数据保存机制

  ### 保存时机

  本系统提供**两种保存方式**：

  | 方式     | 操作      | 说明                        |
  | :------- | :-------- | :-------------------------- |
  | 手动保存 | 菜单选项6 | 即时将内存数据写入磁盘      |
  | 退出保存 | 菜单选项8 | 程序退出前自动调用 `save()` |

  ### 重要警告

  **数据在保存前仅存在于内存中！**

  - **切勿强制关闭程序**（如点击控制台 × 按钮、任务管理器结束进程、断电等）
  - **切勿使用 `Ctrl+C(^C)` 等中断信号终止程序**
  -  **建议在进行重要操作后手动选择选项6保存数据**
  -  **必须通过菜单选项8正常退出**，确保数据被完整写入磁盘

  ### 强制终止的后果

  如果在操作后未保存且未正常退出，**本次运行期间的所有修改将全部丢失!**

------

## 八、注意事项

  1. **手动保存习惯：** 建议在完成重要操作后使用选项**6**手动保存，避免意外导致数据丢失
  2. **库存管理：** 借书时自动减库存，还书时自动加库存；删除借阅记录不会自动恢复库存**(不修改文件可以忽略)**
  3. **学生管理：** 借书时自动记录学生信息，同一学号不会重复创建
  4. **编号唯一性：** 图书编号和学生学号作为唯一标识，重复编号将不被创建
  5. **日期格式：** 输入日期时使用空格分隔，格式为 `年 月 日`（如 `2026 4 29`）
  6. **帮助文档：** 如对操作有疑问，可随时选择选项7查看本帮助文档
  7. **数据备份：** 如需备份，可直接复制 `books.txt`、`students.txt`、`records.txt` 三个文件
  8. **不建议手动操作Text文件**，如需操作，请确保不会出现Id重复等情况。
